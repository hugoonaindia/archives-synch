Tema 5 · Sesión 05 · Artefacto 2

El sobreajuste y el jurado rotativo.

Un modelo sobreajustado memoriza los datos de entrenamiento al milímetro, pero se hunde con datos nuevos. Aprendió el libro, no la asignatura. La defensa: partir el dataset en train, val y test, y — cuando los datos escasean — turnar el examen entre todos los trozos con k-fold cross-validation. Aquí lo ves en directo.

El enemigo número uno

El empollón vs el que entiende.

Imagina dos alumnos. Uno se sabe los ejercicios del libro al dedillo, palabra por palabra. El otro no se sabe cada ejercicio, pero ha entendido el concepto. En el examen les ponen problemas parecidos pero distintos.

El empollón · overfitting

Sobreajuste

Memoriza palabra por palabra. En el examen le ponen un problema parecido y se queda en blanco.

Train: 100% · Test: 60%
→ desastre en producción
El que entiende · generaliza

Generalización

No se sabe cada ejercicio, pero ha captado la idea. En el examen razona y resuelve cualquier variante.

Train: 85% · Test: 84%
→ funciona en la vida real
Cómo detectarlo: si el modelo acierta casi todo en entrenamiento pero falla en datos nuevos, tienes sobreajuste. Por eso siempre hay que reservar datos que el modelo nunca haya visto.

Particiones · La regla de oro

  
Nunca examines con las mismas preguntas con las que estudiaste.

  
Partes el dataset en dos (o tres) trozos: con uno entrenas, con el otro examinas. El modelo nunca debe «ver» el conjunto de test hasta el final.

El examen oficial

    
Una sola oportunidad. No se toca hasta el final. Si el modelo «ve» el test mientras entrena, los resultados están inflados.

Sin stratify, podrías acabar con un test que es 100% no-baja por azar. El modelo nunca sería evaluado sobre la clase que de verdad importa.







  
Validación cruzada · k-fold

  
El jurado rotativo.

  
Si tienes pocos datos, partirlos en train/test es un lujo. La validación cruzada k-fold los aprovecha al máximo: cada trozo hace de test una vez. Tienes 5 jueces — en cada ronda, uno actúa de crítico y los otros cuatro entrenan al modelo. Al final promedias las 5 puntuaciones.


  

    
5-fold cross-validation · cada iteración rota el test

from sklearn.model_selection import cross_val_score
from sklearn.svm import LinearSVC

model = LinearSVC(random_state=42)
scores = cross_val_score(model, X, y, cv=5)

print(scores)          # [0.97, 0.93, 0.97, 0.97, 1.00]
print(scores.mean())   # 0.968


    
Qué te dice esto: si las 5 puntuaciones son parecidas, el resultado es estable. Si una fuera 60% y otra 95%, sospechosísimo. Variantes útiles: StratifiedKFold para clasificación · TimeSeriesSplit para series temporales (¡nunca mezclar pasado y futuro!).
