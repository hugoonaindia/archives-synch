¿Qué tipo de aprendizaje
necesita tu problema?

Antes de tocar una sola línea de código, hay una pregunta que decide todo lo demás: ¿qué tipo de aprendizaje encaja con lo que quiero resolver? Aquí lo aprendemos paso a paso.

01 · LA IDEA DE FONDO

Todo modelo de IA tiene tres momentos.

No importa si es un chatbot, un detector de fraude o un sistema de recomendación: todos siguen la misma receta. Datos que entran, un proceso que aprende, una salida que decide.

∂
01 · INPUT
Datos
Recopilas, limpias y entiendes los datos. Sin este paso, el resto no sirve.
→
Σ
02 · PROCESO
Entrenamiento
El modelo busca patrones, ajusta parámetros, minimiza el error. Aquí elegimos algoritmo.
→
ŷ
03 · OUTPUT
Salida
El modelo predice, clasifica, agrupa o decide. La forma de la salida depende del problema.
El error más caro de un proyecto de IA no es elegir mal el algoritmo. Es elegir mal el tipo de aprendizaje.
02 · CUATRO FAMILIAS

Cuatro maneras de aprender.

No hay diez tipos. Hay cuatro grandes formas en que un modelo puede aprender, y cada una sirve para problemas distintos. Pulsa una tarjeta para abrirla.

01 / FAMILIA
Supervisado
Aprende con la respuesta correcta delante. Cada ejemplo del entrenamiento viene con su etiqueta.
¿Cuándo usarlo? Cuando tienes datos históricos etiquetados y quieres que el modelo prediga ese mismo tipo de respuesta para datos nuevos.
  • Regresión: predecir un número (precio, demanda, edad).
  • Clasificación: predecir una categoría (spam/no, sano/enfermo, dígito).
Coste: etiquetar es lento y caro.
02 / FAMILIA
No supervisado
No hay etiquetas. El modelo busca por su cuenta estructura, grupos o relaciones ocultas.
¿Cuándo usarlo? Cuando tienes datos crudos sin respuesta correcta y quieres descubrir qué hay dentro.
  • Clustering: agrupar clientes similares.
  • Asociación: «si compra X, también compra Y».
  • Reducción de dimensionalidad: comprimir cientos de variables en pocas.
Validación: cualitativa, no hay verdad absoluta.
03 / FAMILIA
Semi-supervisado
Pocos datos etiquetados + muchos sin etiquetar. Aprovecha la estructura del segundo grupo para mejorar.
¿Cuándo usarlo? Cuando etiquetar es muy caro (medicina, voz, idiomas) pero tienes datos crudos abundantes.
  • Self-training, pseudo-etiquetado, modelos de consistencia.
  • Base de muchos modelos fundacionales actuales (LLMs, etc.).
Lo mejor de los dos mundos.
04 / FAMILIA
Por refuerzo
Un agente actúa en un entorno y recibe recompensas o castigos. Aprende por prueba y error.
¿Cuándo usarlo? Cuando el problema es secuencial, hay un objetivo a largo plazo y el modelo puede experimentar.
  • Videojuegos, robótica, conducción autónoma.
  • Q-learning, policy gradients, actor-critic, RLHF.
No hay dataset en el sentido clásico: hay un entorno.
03 · LA SALIDA DELATA EL TIPO

Mira la respuesta que esperas: ya sabes qué tipo es.

Una pista práctica: la forma de la salida dice mucho sobre qué aprendizaje encaja. Estos son los patrones más típicos.

salida = NÚMERO
→
Regresión · supervisado · ej. precio de vivienda, demanda de stock, edad estimada
salida = CATEGORÍA
→
Clasificación · supervisado · ej. spam/no spam, dígito 0–9, diagnóstico
salida = GRUPO
→
Clustering · no supervisado · ej. segmentación de clientes, perfiles de usuarios
salida = REGLA
→
Asociación · no supervisado · ej. «pañales → cerveza», cesta de la compra
salida = EJES
→
Reducción de dimensionalidad · no supervisado · ej. visualizar 200 variables en 2D, comprimir antes de entrenar
salida = ACCIÓN
→
Por refuerzo · ej. movimiento de un robot, jugada de ajedrez, anuncio a mostrar
04 · ÁRBOL DE DECISIÓN

Tres preguntas para elegir bien.

Responde tres preguntas sobre tu problema y te diremos qué tipo de aprendizaje deberías usar. Es exactamente el razonamiento que hace un científico de datos cuando empieza un proyecto.

¿Tu modelo va a actuar en un entorno?

Es decir, ¿toma decisiones secuenciales y recibe consecuencias (recompensa o castigo)?

¿Tienes etiquetas en tus datos?

Una etiqueta es la respuesta correcta para cada ejemplo: el precio real, la clase real, el dígito real.

¿Qué forma tiene tu respuesta?

Mira el target del dataset: ¿es un número continuo o una categoría?

RESULTADO · APRENDIZAJE SUPERVISADO

Regresión.

Tienes datos etiquetados y tu salida es un valor continuo. El modelo aprende una función f(x) ≈ y que minimiza el error entre la predicción y el valor real.

Algoritmos típicos:

regresión lineal árbol de decisión random forest SVR red neuronal

Métricas: MAE, RMSE, R².

RESULTADO · APRENDIZAJE SUPERVISADO

Clasificación.

Tienes datos etiquetados y tu salida es una categoría. El modelo aprende a asignar cada nuevo ejemplo a una de las clases que ha visto.

Algoritmos típicos:

regresión logística árbol de decisión random forest SVM k-NN red neuronal

Métricas: accuracy, precision, recall, F1, ROC-AUC.

RESULTADO · APRENDIZAJE NO SUPERVISADO

Sin etiquetas.

No hay respuesta correcta. El modelo busca estructura: grupos, asociaciones o ejes que resuman los datos.

Variantes según lo que busques:

clustering · k-means · DBSCAN asociación · apriori reducción · PCA · t-SNE · UMAP

Validación: cualitativa, con expertos del dominio.

RESULTADO · APRENDIZAJE SEMI-SUPERVISADO

Lo mejor de los dos mundos.

Aprovechas las pocas etiquetas que tienes para guiar al modelo, y la estructura de los muchos datos no etiquetados para mejorar la generalización.

Técnicas:

self-training pseudo-etiquetado consistencia pre-entrenamiento

Casos típicos: medicina, reconocimiento de voz, imágenes médicas.

RESULTADO · APRENDIZAJE POR REFUERZO

Aprender actuando.

Hay un agente que toma acciones, un entorno que cambia y recompensas que guían el aprendizaje. Se modela como un Proceso de Decisión de Markov (S, A, P, R, γ).

Algoritmos típicos:

Q-learning DQN policy gradients actor-critic PPO RLHF

Aplicaciones: videojuegos, robótica, conducción autónoma, ajuste fino de LLMs.

07 · LABORATORIO DE CASOS

Pensamiento crítico en casos reales.

El test rápido te ayuda a memorizar. Pero la realidad llega en forma de párrafos largos, datos a medias y restricciones de negocio. En este laboratorio practicamos lo que realmente harás en una empresa: leer un problema, separar lo que importa, y elegir el enfoque correcto.

CÓMO USAR EL LABORATORIO

Cuatro modos de práctica.

1. Casos narrativos: escenarios empresariales completos. Lee con calma, identifica los datos disponibles, y decide qué tipo de aprendizaje encaja.

2. Detecta el error: situaciones donde alguien aplicó el modelo equivocado. Tu trabajo es averiguar por qué falla.

3. Empareja: casos breves contra familias de algoritmos. Sirve para fijar el vocabulario.

4. Caso completo: un proyecto guiado paso a paso, donde una decisión condiciona la siguiente.

Pulsa primero un caso de la columna izquierda y después la familia de la columna derecha que crees que lo resuelve. Cuando emparejes los seis, pulsa «Comprobar».
