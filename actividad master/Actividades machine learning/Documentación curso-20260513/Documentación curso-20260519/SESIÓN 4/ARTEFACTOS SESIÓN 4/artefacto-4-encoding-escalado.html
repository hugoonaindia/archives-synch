Pro tip · El error que comete todo el mundo

  
Data leakage y pipelines.

  
El error más común al escalar: aplicarlo a todo el dataset antes de partir en train/test. Resultado: información del test se filtra al train. Tu modelo «hace trampa» en validación y luego falla en producción.


  

    

      
✗ Mal · data leakage

      
Escalar todo, después partir


# El scaler ve TODO el dataset
df_scaled = scaler.fit_transform(df)
X_train, X_test = split(df_scaled)

      
Lo que pasa: el scaler aprendió la media y std usando el test. Tus métricas de validación están infladas. Cuando llegue el día del despliegue, dato nuevo no se parecerá a nada que el scaler «conoce».

    

    

      
✓ Bien · fit en train, transform en test

      
El test se trata como dato nuevo


X_train, X_test = split(df)
scaler.fit(X_train)         # aprende solo del train
X_train = scaler.transform(X_train)
X_test  = scaler.transform(X_test)  # aplica lo aprendido

      
Por qué funciona: el test simula dato nuevo. Lo tratas igual que tratarías una fila que llega mañana en producción. Tus métricas son honestas.

    

  


  
La forma profesional: envuélvelo todo en un Pipeline de scikit-learn. Encoder + Scaler + Modelo, en un solo objeto. pipeline.fit(X_train, y_train) hace todo en cadena, sin filtrar test.



from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.compose import ColumnTransformer
from sklearn.linear_model import LogisticRegression

prep = ColumnTransformer([
    ('num', StandardScaler(), ['edad', 'peso', 'cuota']),
    ('cat', OneHotEncoder(), ['plan'])
])

pipe = Pipeline([
    ('preprocesado', prep),
    ('modelo', LogisticRegression())
])

pipe.fit(X_train, y_train)        # todo en cadena, sin leakage
y_pred = pipe.predict(X_test)
