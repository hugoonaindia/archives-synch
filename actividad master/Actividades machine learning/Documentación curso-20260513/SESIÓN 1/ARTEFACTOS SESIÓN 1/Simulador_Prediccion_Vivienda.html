El patron universal de toda IA

Toda IA, sin excepcion, sigue la misma estructura de tres pasos. Aqui lo vas a ver en accion — y vas a poder cambiar los datos para entender cuanto depende el modelo de ellos.

ENTRADA

Datos

Lo que el sistema observa: numeros, texto, imagenes, comportamiento.

→

PROCESAMIENTO

Modelo

Una funcion matematica entrenada que transforma la entrada en salida.

→

SALIDA

Prediccion

Una decision, una clasificacion, un numero, un texto generado.

Nuestro ejemplo: Vamos a predecir el precio de una vivienda a partir de sus metros cuadrados. Es el ejemplo mas clasico en IA porque ilustra perfectamente el patron: un dato de entrada (m2), un modelo que aprende la relacion, y una salida (precio estimado). Pero el patron es universal: cambia las variables y tienes prediccion de abandono escolar, riesgo clinico, recomendaciones, fraude... el esqueleto matematico es el mismo.
Por que esto importa para CCSS: entender este patron en su forma mas simple os da el lenguaje para conversar con cualquier equipo de Data Science. Cuando os digan "entrenamos un modelo", "ajustamos parametros", "evaluamos con MAE", vais a saber exactamente lo que estan haciendo — y, mas importante, sabreis que preguntas hacerles.

Paso 1: Los datos de entrenamiento

Todo empieza con datos. Imaginemos que tenemos un historico de viviendas vendidas en una ciudad. Cada vivienda tiene dos valores: sus metros cuadrados (nuestra entrada, lo que el modelo va a "ver") y su precio de venta (lo que queremos aprender a predecir, llamado tecnicamente la variable objetivo o target).

Observa: a mas metros cuadrados, mayor precio. Existe una relacion lineal entre ambas variables (lo veras claramente en el grafico). Esto es lo que el modelo va a "aprender". En el dataset "Madrid con ruido" la relacion sigue ahi, pero cuesta mas verla — ese ruido es lo que llamamos variabilidad no explicada.

Estos puntos representan nuestros datos. El objetivo del modelo sera encontrar la linea recta que mejor se ajuste a estos puntos. A eso se le llama regresion lineal, el modelo mas simple de Machine Learning. Aunque sea simple, su logica es la misma que usa un modelo de deep learning con millones de parametros: ajustar parametros para minimizar el error.

Paso 2: Entrenamiento del modelo

Ahora viene la parte central. El modelo va a aprender la relacion entre metros cuadrados y precio. Para ello, ajustara dos parametros internos:

La ecuacion del modelo: Precio = pendiente x m2 + base

pendiente (m): cuanto sube el precio por cada metro cuadrado adicional. Es la "tasa de cambio".
base (b): el precio "base" cuando los metros son cero (un ajuste matematico, no tiene interpretacion fisica).

Tecnicamente: y = mx + b — la ecuacion mas famosa de la estadistica, escondida en la base de practicamente cualquier modelo predictivo.

El entrenamiento consiste en ajustar estos dos numeros iterativamente para que la linea se acerque lo maximo posible a los datos reales. Cada iteracion se llama una epoca (epoch). En cada epoca:

  1. El modelo hace predicciones con los parametros actuales.
  2. Calcula el error comparando predicciones con valores reales (lo llamamos funcion de coste o loss).
  3. Ajusta los parametros un poquito en la direccion que reduce el error.
  4. Repite. Y repite. Hasta que el error deja de bajar.

La metrica de error que usamos aqui es MAE (Mean Absolute Error): la diferencia media absoluta entre lo que el modelo predice y el valor real. Vas a verla bajando epoca a epoca en el grafico de la derecha — es la "evidencia" de que el modelo esta aprendiendo.

Que ha pasado? El modelo ha encontrado la recta que minimiza la distancia a todos los puntos. Las metricas que ves son las que cualquier data scientist te ensenaria al evaluar un modelo: MAE (error medio en euros) y R2 (proporcion de varianza explicada — entre 0 y 1, mas alto = mejor). Ahora puede usar esta recta para predecir el precio de una vivienda que nunca ha visto.

Paso 3: Prediccion

El modelo ya esta entrenado. Ahora puedes introducir los metros cuadrados de una vivienda nueva y el modelo predecira su precio. Esto es lo que pasa cada vez que un sistema de IA "decide" algo en produccion: aplica la formula que aprendio a una entrada nueva.

Experimenta: que pasa si cambias los datos?

Vuelve al Paso 1, cambia de ciudad y reentrena. Veras dos cosas:

  1. La pendiente cambia (Paris y Tokio son mas caros, asi que cada m2 vale mas).
  2. El R2 baja con el dataset "ruidoso" aunque el modelo siga "funcionando". El error medio (MAE) tambien sube.

Eso ilustra una idea profunda: el modelo no aprende "lo que es verdad", aprende lo que sus datos le ensenan. Si los datos son sesgados o ruidosos, el modelo lo sera tambien. Esto es el "Garbage In, Garbage Out" del que hablamos en clase, y es la razon por la que vuestro juicio sobre la calidad y representatividad de los datos es estrategico.

Reflexion para CCSS: Si entrenaras un modelo para predecir el riesgo de abandono escolar, tu "m2" serian variables como asistencia, calificaciones, participacion... y tu "precio" seria el riesgo (alto/medio/bajo). El patron es identico. Lo que cambia es el contexto — y ahi es donde vuestra experiencia como profesionales del comportamiento humano es insustituible.

Lo que llevarte de este simulador: ahora cuando alguien te diga "entrenamos un modelo de prediccion", sabes que estan haciendo. Sabes que el modelo es una funcion matematica, que la calidad de los datos define los limites de lo que puede aprender, y que cada prediccion individual es una aplicacion de la formula aprendida. Y, sobre todo, sabes que las decisiones humanas — que datos recoger, como etiquetarlos, como interpretar los resultados — son las que realmente determinan si la IA es util o danina.
Para profundizar mas alla de la regresion lineal: los modelos modernos (random forests, redes neuronales, transformers) son matematicamente mas complejos pero conceptualmente identicos: ajustan miles o millones de parametros para minimizar una funcion de coste sobre datos de entrenamiento. Lo que cambia es la flexibilidad del modelo (puede aprender relaciones no lineales) y la cantidad de datos que necesita.
